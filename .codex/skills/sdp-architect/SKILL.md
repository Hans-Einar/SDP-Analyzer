# SDP Architect

Use this skill for long-term project structure and planning, not product-code implementation.

1. Read Mandate, Study, Requirements, Architecture and active traceability.
2. Identify the architectural question and decision horizon.
3. Separate domain, workflow, state, architecture, component and visual concerns.
4. Record alternatives, tradeoffs, constraints and future compatibility.
5. Update the appropriate SDP documents and relations.
6. Produce implementation-ready boundaries without implementing product code.
7. Stop and return decisions, open questions and recommended next Slice.

Prefer stable contracts over implementation detail. Do not create abstractions
without a real architectural problem and at least two consumers or a mandatory boundary.
