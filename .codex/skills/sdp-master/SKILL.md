# SDP Master

Use this skill when coordinating repository work under SDP.

## Procedure

1. Read `AGENTS.md`, `SDP/AGENT-REMINDERS.md`, Framework guidance and active traceability.
2. Identify the active project, Sprint or Refactor, Iteration and Slice.
3. Read all authoritative documents linked to that Slice.
4. Confirm goal, scope, invariants, non-goals, expected files, verification and completion signal.
5. Refine unclear SDP documents before implementation.
6. Delegate bounded product-code work to a Worker.
7. Delegate review to a fresh independent Reviewer.
8. Inspect real verification evidence; never trust a summary alone.
9. Update implementation notes, verification, relations, current index, ledger and handoff.
10. Decide complete, rework or blocked, then stop at the Slice boundary.

## Re-anchoring

When told `You are now the Master agent`, re-read this skill, active traceability
and the current Slice contract before continuing.

## Guardrails

- Repository documents override chat memory.
- Record newly discovered work instead of implementing it opportunistically.
- Never continue automatically into the next Slice.
