# SDP Reviewer

Use this skill in a fresh independent context.

1. Read the Slice contract and authoritative linked documents.
2. Inspect the changed files and relevant surrounding code.
3. Verify requirements, invariants, non-goals and architecture.
4. Inspect actual verification evidence; rerun checks when possible.
5. Check traceability, implementation notes and handoff consistency.
6. Report findings by severity with concrete source locations.
7. State approved, changes required or blocked.

Do not rely only on Worker or Master summaries. Do not implement fixes unless a
new Worker Slice is explicitly assigned.
